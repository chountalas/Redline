UX flow redesign audit

Redline becomes a document review workspace.

The product should no longer feel like a one-off lease checker wrapped in a Mac shell. It should feel like a reusable review system with explicit profiles, cited evidence, and a lease-math profile as the first production-grade lane.

Significant redesign

Why this verdict: Redline already has a credible native workspace, real loading/error/empty states, source evidence, review notes, export, and provider plumbing. The limiting issue is the product contract: the real engine, CLI, MCP tool, and much of the old UI language made the workflow lease-specific.

Executive read

The app was wider than the engine.

Redline's sample library already implied a broader contract-review product, with sample leases, an MSA, and an NDA. The real run path still converted every result into a lease check, while the CLI and MCP entry points exposed only redline check and check_lease. The redesign therefore separates the durable workspace from the active production profile.

Keep

The cited review surface, source sync, deterministic findings, advisory separation, note-taking, export, provider settings, and local-first storage are the product's strongest assets.

Fix now

Stop presenting the Mac app as lease-only. Rename the intake, empty state, run controls, verdict copy, export memo language, and docs around documents, comparison context, approval, and active profiles.

Build next

Add a profile registry so lease math, vendor contracts, NDAs, renewals, invoices, and custom checklists can share one workspace without pretending the current lease extractor handles all of them.

Design DNA

Redline is not a chat app. It is a cited decision desk.

The product should make a reviewer faster and more confident by showing what was checked, what evidence supports it, and what blocks approval. AI can extract and suggest, but the interface should keep deterministic checks visually and conceptually separate from advisory review.

Evidence first

Every meaningful finding should link back to source text, clause, or page.

Profile explicit

A review must say what rule profile ran and what it does not cover.

Approval language

Use "approval" and "review" rather than lease signing language across generalized flows.

Quiet operations

Provider, key, export, and storage choices should stay available without dominating document intake.

Current-state map

The workspace is real, but the engine seam is still narrow.

The updated app now opens as a document-review shell with profile-aware execution. The default path runs the broader general-lease profile, while the original lease-math lane remains available for per-display-face validation.

One idea

Make profiles the product boundary.

The next architecture move should be a first-class review profile abstraction. A profile owns the extractor schema, comparison-sheet schema, deterministic rules, advisory prompt, empty-state examples, and export vocabulary. The workspace owns intake, source evidence, notes, review state, provider settings, and export mechanics.

Design principle: Every screen may say "document", but every result must say exactly which profile checked it. Generic language should widen the door, not obscure the limits of the active rule engine.

Reframed navigation

Use a general workspace with profile-aware metadata.

The main navigation should organize by review work, not by lease workflow. The current native structure is close; the remaining UX move is to expose the active profile as metadata on the document and run sheet rather than making the whole app lease branded.

Safety net

Edge cases checked in the redesign pass.

The generalized copy pass should not regress resilience. The current app already handles many operational states; future non-lease profiles need additional limits and recoverability messaging.

What changed in v2

The implemented pass widens the shell without lying about the engine.

The branch now presents the native app as document review, not lease-only intake. It keeps the real active profile named in result metadata and documentation.

Generalized now

Home, empty state, toolbar help, run sheet, drop copy, comparison context, focus placeholder, preflight messages, source fallback, verdict copy, report wording, export memo wording, and privacy docs.

Still intentionally specific

The production extractor, rules, CLI command, and MCP tool are still commercial lease validation. They should stay explicit until a profile registry ships.

Roadmap

Generalization should land in three steps.

Do not jump straight to "any document" marketing. Ship generalized structure while adding profiles that earn their names.
